Angular Routing

Implement routing

In this section we will implement the following menu structure with routing:

   AppComponent
      +----------- HomeComponent
      +----------- FlightSearchComponent
      +----------- PassengerSearchComponent (Dummy)

Respect the following aspects:

  • The AppComponent and HomeComponent is part of AppModules
  • The two other components are part of FlightBookingModules
  • Each module gets its own router config with forRoot or forChild assigned.

Follow the steps below to implement this:

  1. Add the following additional (dummy-)component, which should be displayed as a separate route:

    • HomeComponent in src/app/home
    • PassengerSearchComponent in src/app/flight-booking/passenger-search

  2. Make sure that the HomeComponent is registered in AppModule.

  3. Also make sure that the PassengerSearchComponent is registered in FlightBookingModule.

  4. Create a router config in app.routes.ts which is used in the older src/app in the file AppModule.

    Show code 

    export const APP_ROUTES: Routes = [
  {
    path: '',
    redirectTo: 'home',
    pathMatch: 'full'
  },
  {
    path: 'home',
    component: HomeComponent
  },
  {
    path: '**',
    redirectTo: 'home'
  }
];

  1. Open app.module.ts and import RouterModule from @angular/router. Use the router config from the file app.routes.ts.

    Show code 

    @NgModule({
  imports: [
    […]
    RouterModule.forRoot(APP_ROUTES) // <-- Add this line!
  ],
  […]
})
export class AppModule { }

    Notice that the method forRoot is used. This is done because AppModule is our main/root module.

  2. Create a file flight-booking.routes.ts in the folder src/app/flight-booking with a routing configuration for FlightSearchComponent and PassengerSearchComponent.

    Show code 

    export const FLIGHT_BOOKING_ROUTES: Routes = [
  {
    path: 'flight-search',
    component: FlightSearchComponent
  },
  {
    path: 'passenger-search',
    component: PassengerSearchComponent
  }
];

  3. Open the file flight-booking.module.ts in the folder src/app/flight-booking and import the RouterModule. Use the newly created router config. Notice that we now have to use forChild to register the router config for a Child-Module (Feature-Module).

    Show code 

    @NgModule({
  imports: [
    [...]
    RouterModule.forChild(FLIGHT_BOOKING_ROUTES) // <-- Add this line!
  ],
  [...]
})
export class FlightBookingModule { }

  1. Open the file app.component.html and replace the element/component flight-search with <router-outlet></router-outlet>. This is the placeholder to render the route components in.

    Show code 

    <div class="content">
    <!-- <flight-search></flight-search> --> <!-- old -->
    <router-outlet></router-outlet> <!-- new -->
</div>

  2. Open the file sidebar.component.html and extend the menu items with the directive routerLink. This directive activates the related routes on click.

    Show code 

    <li class="nav-item">
    <a class="nav-link" routerLink="passenger-search">
        <em class="fa fa-users"></em>
        Passenger
    </a>
</li>
<li class="nav-item">
    <a class="nav-link" routerLink="flight-search">
        <em class="fa fa-plane"></em>
        Flights
    </a>
</li>
<li class="nav-item">
    <a class="nav-link" routerLink="home">
        <em class="fa fa-ticket"></em>
        Home
    </a>
</li>

  3. Test the solution.

Bonus: Routing with the Hash-Fragment and Tracing *

To change how the router works there is a second argument in the forRoot method. In this object you can define to use Hash-Fragment in the url instead of the history api (i. e. http://localhost:4200/#/route instead of http://localhost:4200/route). There is also an option to display tracing messaged in the console:

RouterModule.forRoot(APP_ROUTES, { useHash: true, enableTracing: true});

Activate the options and make sure that the # is placed in the url and the logs are in the console.

Parameters

In this exercise we will (if not done in another exercise) create a FlightEditComponent component in FlightBookingModule:

   AppComponent
      +----------- HomeComponent
      +----------- FlightSearchComponent ---- id ----+
      +----------- PassengerSearchComponent          |
      +----------- FlightEditComponent [NEU!] <------+

This should take a parameter id and an optional parameter showDetails and for now just display them. The component should be displayed when clicking on an edit button in the FlightCardComponents.

  1. Create the FlightEditComponent in src/app/flight-booking/flight-edit and register it.

  2. Inject ActivatedRoute in the constructor of FlightEditComponent and read the id and showDetails.

    Show code 

    @Component({
  selector: 'app-flight-edit',
  templateUrl: './flight-edit.component.html'
})
export class FlightEditComponent implements OnInit {
  id: string;
  showDetails: string;

  constructor([...], private route: ActivatedRoute) { }

  ngOnInit() {
    this.route.params.subscribe(
      params => {
        this.id = params['id'];
        this.showDetails = params['showDetails'];
      }
    );
  }

  [...]
}

  3. Open flight-edit.component.html and display the url params.

    Show code 

    <div class="card">
  <div class="header">
    <h1 class="title">Flight Edit</h1>
  </div>
  <div class="content">
    <p>Id: {{id}}</p>
    <p>ShowDetails: {{showDetails}}</p>
  </div>

  [...]

</div>

  4. Open the file flight-booking.routes.ts and add a new route for FlightEditComponent.

    Show code 

    export const FLIGHT_BOOKING_ROUTES: Routes = [
  […],
  {
    path: 'flight-edit/:id',
    component: FlightEditComponent
  }
];

    The segment :id is used as a placeholder for id. There is no placeholder for showDetails which means it is an optional url parameter (matrix parameter).

  5. Open the file flight-search.component.html and add a link that navigates to the flight-edit page.

    Show code 

    <a class="btn btn-default" 
  [routerLink]="['/flight-edit', f.id, { showDetails: true }]">   
  Edit
</a>

  6. Test the solution: Search flights and click the edit link on one of the flights.

Bonus: Flight Edit *

In this section we will enable editing of a flights in FlightEditComponent.

  1. Open flight.service.ts and implement a method named findById which fetches a flight by its id.

    To fetch a flight use the HttpClient and its get method with the param id.

    If not already implemented use the post method of HttpClients to save a flight:

    return this.http.post<Flight>(url, flight, { headers });

    Notice that the flights with id's 1 to 5 are not editable.

    Show code 

    findById(id: string): Observable<Flight> {
    const url = 'http://www.angular.at/api/flight';
    const params = new HttpParams()
                      .set('id', id);
    return this.http.get<Flight>(url, { params});
}

  1. In FlightEditComponent load the flight with the id from the url. On every url parameter change, load the new flight.

    Show Code 

    @Component({
  selector: 'app-flight-edit',
  templateUrl: './flight-edit.component.html'
})
export class FlightEditComponent implements OnInit {
  
  […]

  ngOnInit() {
    this.route.params
    .pipe(switchMap(params => this.flightService.findById(params.id)))
    .subscribe(
      (flight: Flight) => { this.selecteFlight = flight; this.message = ''; },
      err => { this.message = 'Error'; }
    );
  }
}

  2. Open flight-edit.component.html and make sure that the flight with the id in the url is loaded.

  3. Test your solution.
